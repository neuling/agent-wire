The problem

Two Claude Code sessions, one project, zero coordination.

You run one session on the frontend, another on the backend. They have no idea the other exists. Backend changes an API response. Frontend silently breaks. You become the messenger, copy-pasting context between tabs like it's 2004.

backend-agent (Claude A) "Updated GET /users/:id — avatar_url is now returned as nullable."
frontend-agent (Claude B) "Refactored UserProfile to use the existing schema. Lgtm!" (it is not, in fact, lgtm)
you, sighing alt-tab alt-tab alt-tab

How it works

A local daemon. A thin stdio bridge per session. Live push into context.

agent-wire runs a single daemon on 127.0.0.1:4747. Every Claude Code session spawns a tiny stdio MCP bridge that declares the claude/channel capability. When another agent sends you something, it appears inside your context as a live <channel> tag — in real time, while you're mid-thought, without anyone asking for it.

→

Push into context

Items arrive as native <channel> tags via the Channels API. No polling, no waiting. Pull + piggyback fallback for other MCP clients.

◆

Project cards

Each agent publishes a short card — CLAUDE.md summary, repo, branch, key deps. Others read it via wire_describe without touching your files.

●

Live dashboard

Open localhost:4747 to see agents, an activity feed, and the shared decisions log — streaming live over SSE.

Dashboard

See every agent, every note, every decision — live.

The daemon serves a three-column view at http://127.0.0.1:4747. Agents panel on the left, activity feed in the middle, shared decisions log on the right. Zero config, no login, no setup — just open the URL while the daemon is running.

Every wire_* call updates it live. Great as a selling demo, better as a debugging console.

● frontend-agent  Refactoring UserProfile…
● backend-agent  Adding avatar_url to GET /users
→ backend → frontend: "avatar_url is nullable"
◆ frontend: "Updating TypeScript types"
📋 log: "API contract locked"

Install

One line. Then launch your sessions.

Adds agent-wire to Claude Code at user scope. The bridge lazy-starts the daemon on first use — no separate service to manage. 

$ claude mcp add --scope user agent-wire -- npx -y agent-wire-bridge
# then launch a session with channel push enabled:
$ wire-claude

Not yet on npm. Until it is, clone the repo and replace npx -y agent-wire-bridge with the absolute path to your local dist/bridge/index.js. Full install, permissions, and CLAUDE.md instructions live in the README.

Scope

Loopback only. Single user. In-memory. By design.

agent-wire binds only to 127.0.0.1. It makes no outbound connections. Nothing persists to disk. There are no webhooks, no remote relay, no LAN. Every item you send is visible only to agents running as your user on this machine, for the lifetime of the daemon.

Not a goal: persistence, auth, orchestration, file sync, multi-machine. Those belong in different tools.
